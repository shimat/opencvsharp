Corrections & caveats

  • Several parallel research passes initially reported FaceRecognizerSF, QRCodeEncoder, and aruco's Board/GridBoard as "already wrapped." Direct verification against src/OpenCvSharp/Modules/ found none of the four exist — they're listed below as real gaps.
  • mcc (Macbeth color-checker detection) showed up as a candidate in objdetect, but cmake/opencv_build_options.cmake explicitly sets BUILD_opencv_mcc OFF — it isn't compiled into this project's native build at all, so it's excluded below rather than listed as a gap.
  • freetype and alphamat only build on macOS today — their dependencies (freetype2/harfbuzz, Eigen3) are vendored in vcpkg.json for osx only, not Windows/Linux.
  • CUDA modules, viz (VTK), sfm (Ceres/gflags/glog), cvv (Qt), ovis (OGRE), and the MATLAB/Julia bindings are out of scope everywhere — none of those toolchains are vendored on any CI platform.

Net-new modules worth adding

These aren't wrapped at all today, but compile cleanly with the project's existing dependency set — no new vcpkg packages, no CI changes.

rgbd linemod template-matching detector plus the KinFu / DynaFu / ColoredKinFu / LargeKinfu volumetric-fusion family — all Algorithm-derived with Params structs, same shape as the recently-wrapped ptcloud classes. Needs geometry + ptcloud (both already wrapped); OpenGL/viz interop is optional only. Large
xstereo QuasiDenseStereo — dense stereo matching on top of the already-wrapped stereo + tracking modules. Medium
hfs HfsSegment real-time superpixel/hierarchical segmentation. CPU path has zero hard dependency (CUDA is optional-only, unlike older OpenCV versions). Small–medium
plot Plot2d — draws a line/scatter plot straight from a Mat. Pure core+imgproc, no dependency risk. Small–medium
phase_unwrapping HistogramPhaseUnwrapping for structured-light / interferometry pipelines. No dependency risk. Small–medium
signal One free function, resampleSignal. Trivial, zero dependencies. Small

Also considered and rejected: xobjdetect (only symbol is WBDetector, plain CV_EXPORTS not CV_WRAP — duplicates functionality already covered by CascadeClassifier/HOGDescriptor).

Quick wins

Small, self-contained additions across existing modules — a native bridge function plus a thin C# method or property each, no new class scaffolding.

core

TickMeter

Complete timing-utility class (start/stop/getFPS/…) with zero wrapper presence today — commonly wanted for benchmarking.

superres

SuperResolution.OpticalFlow

Native bridge and P/Invoke are already wired; the C# property body is just commented out as a TODO.

imgproc

StackBlur, ThresholdWithMask

Two small OpenCV-5 free functions with no C# entry point yet.

calib

FixS1S2S3S4 wrong value

CalibrationFlags.cs defines it as a duplicate of ThinPrismModel (0x08000) instead of the native 0x10000 — a silent correctness bug, not just a gap. (Fix in progress.)

barcode

Missing getters

GetDownsamplingThreshold/GetDetectorScales/GetGradientThreshold — setters exist, getters don't, on both sides.

aruco

RefineDetectedMarkers

Native bridge function already exists; no C# method calls it.

img_hash

Six one-shot free functions

averageHash, pHash, blockMeanHash, etc. — the classes are fully wrapped, just not the convenience call-without-instantiating form.

wechat_qrcode

ScaleFactor accessor

GetScaleFactor/SetScaleFactor missing entirely.

shape

Two missing getters

ShapeContextDistanceExtractor.GetCostExtractor/GetTransformAlgorithm — setters exist, getters don't.

flann

Index.Load / Release

Only Save is wrapped; there's no way to load a previously-saved index or explicitly release one.

Stitching pipeline

The high-level Stitcher class covers the common panorama-stitching path end to end. Its internal detail:: namespace — the pieces you'd swap out to build a custom pipeline — is largely unwrapped, similar in spirit to dnn's Layer or ml's TrainData: likely a deliberate scope line rather than an oversight, but a real gap for anyone wanting fine control.
